Detailed Description

High-level controller for the event-driven simulation engine.

Acts as the single entry point for running experiments. It initializes traffic models, schedules events, consumes them in chronological order, updates aggregate statistics, and persists results. The simulator consumes SimulationParameters and produces CSV/summary outputs plus queue analytics. It collaborates with EventQueue, SimulationClock, StatisticsCollector, and IO helpers.

void core.Simulator.run ( )

Runs the simulation loop and exports data to organized outputs.

The method drains the EventQueue while the SimulationClock is below totalSimulationTime. For each event it updates the corresponding source, schedules the next event, captures samples, and writes to the OutputSink. At completion the method finalizes queue statistics and exports CSV plus summaries.
